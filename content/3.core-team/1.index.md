## Izin Dan Persetujuan MainTainer

Sebelum anda meninginkan domain email custom, pastikan anda sudah mendapatkan izin dan persetujuan dari MainTainer. Hal ini penting untuk memastikan bahwa penggunaan email milik perusahaan sesuai dengan kebijakan dan standar yang berlaku. kami
pastikan anda sudah mendapatkan izin dan persetujuan dari Maintainer kami. anda bisa mengajukan di github ['issue'](https://github.com/nlfts/nlfts/issues) atau melalui
github ['discussions'](https://github.com/nlfts/nlfts/discussions). dan satu satunya hal yang tidak boleh anda lakukan adalah meminta request nama domain melalu email resmi.
semua nama domain harus bersifat umum dan tidak boleh untuk di sembunyikan. 


## Cara Meminta Domain Email Melalui Github

### Melalui Issue

#### 📋 Prosedur Pengajuan

1. **Buka Issue Baru**: Masuk ke repositori organisasi NLFTs dan pilih menu **[Issues*](https://github.com/nlfts/nlfts/issues)**.
2. **Gunakan Format Judul**: Gunakan format `Request(domain) : Nama Lengkap` (Contoh: `Request(domain) : Nairha Saqinana`).
3. **Isi Data Diri**: Lengkapi formulir sesuai template di bawah ini.
4. **Tunggu Label**: Setelah terkirim, Maintainer akan memberikan label **`Request`** pada Issue tersebut.
5. **Diskusi & Verifikasi**: Maintainer mungkin akan menanyakan beberapa detail di kolom komentar sebelum melakukan aktivasi.

---

#### 📝 Template Pengajuan (Copy-Paste)

Gunakan format di bawah ini saat mengisi deskripsi Issue:


```markdown
#### 📧 ['Meminta'] pengajuan domain email perushaan

**Nama Lengkap:** [Isi Nama Anda]

**Username GitHub:** @username

**Alamat Email yang Diinginkan:** nama_pilihan@nlfts.dev

**Tujuan Penggunaan:**
- [ ] Komunikasi Internal Team
- [ ] Pribadi
- [ ] Kebutuhan Komunitas
- [ ] Lainnya: ______

**Catatan Tambahan:**
(Opsional) Sertasikan informasi tambahan jika diperlukan.

```


---

#### Alur Setelah Submit

* **Labeling**: Issue Anda akan otomatis ditandai dengan label `Request`.
* **Review**: Tim infrastruktur akan memeriksa ketersediaan *username* tersebut.
* **Setup**: Jika disetujui, Anda akan menerima instruksi aktivasi dan password sementara melalui jalur komunikasi privat (DM/Email Pribadi).
* **Closing**: Issue akan ditutup (**Closed**) setelah email berhasil diaktivasi dan dikonfirmasi oleh pengguna.

---

#### 💡 Tips Keamanan

* Jangan pernah menuliskan **Password** yang Anda inginkan di dalam GitHub Issue karena bersifat publik.
* Pastikan akun GitHub Anda sudah mengaktifkan **[Two-Factor Authentication (2FA)](https://docs.github.com/en/authentication/securing-your-account-with-two-factor-authentication-2fa)** sebelum mengajukan request ini.
* Kami juga akan menilai anda dari aktivitas dan kontribusi di dalam github sekecil apapun itu, jadi pastikan anda aktif di dalam github untuk mendapatkan kepercayaan dari Maintainer.
